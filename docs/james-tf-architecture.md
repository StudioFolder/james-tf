# james.tf
## Website Architecture & Development Reference
*March 2026 — StudioFolder × James Taylor-Foster*

---

## 1. Overview

This document describes the architecture, file structure, tooling, and deployment workflow for the james.tf website — currently in its holding page phase, with the full site to follow. It serves as a reference for ongoing development.

The project is built with plain HTML and CSS (with minimal vanilla JavaScript), using a lightweight build step to separate content from markup. The stack is intentionally minimal, prioritising performance, legibility, and ease of client-side content editing.

---

## 2. Technology Stack

### Holding Page

- **HTML + CSS + vanilla JS** — no framework, no build framework
- **marked.js** — Markdown parser (Node.js, build time only)
- **Instrument Sans** — served via Google Fonts
- **Node.js v23** — runs the build script locally and on Vercel

### Full Website (planned)

## 4. File Structure

```
james-tf/
├── template.html       ← HTML shell, never edited directly after build
├── index.html          ← generated by build.js, do not edit manually
├── style.css           ← all visual styles
├── build.js            ← build script: reads content.md → generates index.html
├── content.md          ← all editable content (client edits this file)
├── deploy.sh           ← convenience script: build + commit + push
├── vercel.json         ← tells Vercel to run npm run build on deploy
├── package.json        ← Node project config, defines build script
├── package-lock.json
├── node_modules/       ← gitignored
├── assets/             ← images (portrait photo, etc.)
├── docs/               ← project documentation
└── .gitignore          ← ignores node_modules
```

### Key Principle

The client (James Taylor-Foster) **never edits** `index.html` or `template.html` directly. All content changes happen exclusively in `content.md`. The build script regenerates `index.html` from the template and the content file on every deploy.

---

## 5. Build System

### How it works

The build script (`build.js`) performs the following steps:

1. Reads `template.html` — the structural HTML shell with `{{NAV}}` and `{{CONTENT}}` placeholders
2. Reads `content.md` and splits it into sections by `# Heading` markers
3. For each section, generates a nav button (left column) and a content panel (right column)
4. Injects the generated HTML into the placeholders and writes `index.html`

### content.md structure

Sections are defined by top-level Markdown headings (`# Title`). The section title becomes both the nav label and the panel ID. All standard Markdown formatting is supported within sections.

```markdown
# Projects

2025–27 *Project Title* – Institution

# Writings

[*Article Title*](https://url.com), Publication
```

### Running the build

To build locally (rebuilds `index.html`, no push):

```bash
npm run build
```

To build, commit and push to GitHub (triggers Vercel redeploy):

```bash
./deploy.sh "commit message"
```

### Vercel configuration

The `vercel.json` file instructs Vercel to run `npm run build` before each deployment, so the content pipeline works identically in CI as it does locally.

## 6. Client Content Editing

James Taylor-Foster can update all page content autonomously, without any technical intervention from StudioFolder, using the GitHub web editor:

1. Go to `github.com/StudioFolder/james-tf`
2. Open `content.md` and click the pencil (Edit) icon
3. Make changes using standard Markdown syntax
4. Click "Commit changes" — Vercel redeploys automatically within ~30 seconds

*Note: the client currently pushes directly to `main`, which deploys immediately to the live site. This can be changed to a separate branch with a pull request flow if a review step is desired.*

## 7. Design System (Current)

### Typography

- **Typeface:** Instrument Sans (Google Fonts)
- **Base size:** `0.8rem`
- **Line height:** `1.4`
- **Weight:** 400 throughout
- **Case:** uppercase for all UI elements
- **Letter spacing:** `0.01em`

### Colour

- **Background:** `#f0f0f0` (`--color-bg`)
- **Text:** `#1a1a1a` (`--color-text`)

### Layout

- **Left column offset:** `--left: 12rem`
- **Top offset:** `--top: 1.5rem`
- **Face offset:** `calc(--left - 3.5rem)`
- **Right column:** begins at `33vh` vertically

### Animations

- **Fade-in cascade:** name → sections → footer, each 0.3s apart, 0.6s duration
- **Eye blink:** JS-driven, both eyes sync, random interval 2–6s
- **Section panels:** opacity + visibility transition on hover, 0.3s ease
- **Link underline:** scaleX from right to left, 0.2s linear

### Mobile (≤768px)

- Face: fixed top right
- Name: fixed top left, reduced left margin (`1.75rem`)
- Sections and right column: hidden
- Footer: fixed bottom left, full width minus margins
