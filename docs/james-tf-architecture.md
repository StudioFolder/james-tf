# james.tf
## Website Architecture & Development Reference
*March 2026 — StudioFolder × James Taylor-Foster*

---

## 1. Overview

This document describes the architecture, file structure, tooling, and deployment workflow for the james.tf website — currently in its holding page phase, with the full site to follow. It serves as a reference for ongoing development.

The project is built with plain HTML and CSS (with minimal vanilla JavaScript), using a lightweight build step to separate content from markup. The stack is intentionally minimal, prioritising performance, legibility, and ease of client-side content editing.

---

## 2. Technology Stack

### Holding Page

- **HTML + CSS + vanilla JS** — no framework, no build framework
- **marked.js** — Markdown parser (Node.js, build time only)
- **Instrument Sans** — served via Google Fonts (400, italic 400)
- **ABC Diatype Mono Condensed** — loaded locally via `@font-face` (Regular + Italic OTF), used for bio text and entry years
- **Node.js v23** — runs the build script locally and on Vercel

### Full Website (planned)

- **SvelteKit** — component framework, SSR + static generation
- **TBD CMS** — to be decided; will feed content to SvelteKit

## 5. Build System

### How it works

The build script (`build.js`) performs the following steps:

1. Reads `template.html` — the HTML shell with `{{NAV_PRIMARY}}`, `{{NAV_SECONDARY}}`, `{{NAME_DELAY}}`, `{{BIO_DELAY}}`, `{{BIO_CONTROLS_DELAY}}`, `{{FOOTER_DELAY}}`, `{{FOOTER}}`, `{{BIO_SHORT}}`, `{{BIO_MEDIUM}}`, `{{BIO_LONG}}`, `{{BIO_SHORT_WORDS}}`, `{{BIO_MEDIUM_WORDS}}`, `{{BIO_LONG_WORDS}}` placeholders
2. Reads `content.md` and splits it into sections by heading level (`#` for primary nav, `##` for secondary nav and special sections)
3. Detects special sections by slug and routes them appropriately
4. For each nav section, generates a `.nav-row` containing a nav button (with `aria-expanded="false"`) and its corresponding section panel
5. Calculates staggered animation delays for the full fade-in cascade (name → bio → bio-controls → primary nav items → secondary nav items → footer)
6. Calculates word counts for each bio version
7. Injects all generated HTML into the placeholders and writes `index.html`

### Custom Markdown renderer

The build script uses a custom `marked` paragraph renderer. Each line within a paragraph is examined in order:

1. **Portrait entries** — lines matching `## | path | caption` (2-digit number followed by `|`) are wrapped in `<span class="entry-portrait">` with `data-id`, `data-src`, and child elements: `.entry-trigger` ("View"), `.entry-download` (download link), and `.entry-caption` (hidden caption text)
2. **Chronological entries** — lines matching `year | content` (4-digit year followed by `|`) are wrapped in `<span class="entry">` with `.entry-year` and `.entry-text` spans
3. **Full-width text** — all other lines are wrapped in `<span class="entry-full">`

### content.md structure

Sections are defined by Markdown headings:

- `#` — primary nav section (upper nav block)
- `##` — secondary nav section (lower nav block) OR special section (detected by slug)

Special `##` sections are excluded from the nav and injected elsewhere:

| Slug | Purpose |
|---|---|
| `footer` | Injected into the `<footer>` element |
| `bio-short` | Short bio version |
| `bio-medium` | Medium bio version |
| `bio-long` | Long bio version |

```markdown
## bio-short

James Taylor-Foster is a curator, editor, and writer.

## bio-medium

James Taylor-Foster is a curator, editor, and writer. [...]

## bio-long

James Taylor-Foster is a curator, editor, and writer. [...]

# Projects

2025-27 | *Project Title* – Institution

# Writing

2026 | [*Article Title*](https://url.com), Publication

## Portrait

01 | /assets/images/portrait-01.jpg | Photo by Pierre Björk<br>Stockholm, 2025

## Contact

[email](mailto:hej@james.tf), [instagram](https://www.instagram.com/jtaylorfoster/)

## footer

Unless otherwise stated content on this site [...] licensed under [CC BY-ND 4.0](https://creativecommons.org/licenses/by/4.0/)
```

### Entry formatting

Lines beginning with a year (including ranges like `2025-27`) followed by `|` are formatted as two-column entries — year on the left (set in ABC Diatype Mono), content on the right. All standard Markdown (italics, links) is supported within entries.

Portrait entries use a 2-digit identifier: `01 | /path/to/image.jpg | Caption text`. The caption supports inline HTML like `<br>`.

### Running the build

```bash
npm run build                    # Build locally
./deploy.sh 'commit message'    # Build, commit and push (use single quotes)
```

### Vercel configuration

`vercel.json` instructs Vercel to run `npm run build` before each deployment.

## 6. Client Content Editing

James Taylor-Foster can update all page content autonomously using the GitHub web editor:

1. Go to `github.com/StudioFolder/james-tf`
2. Open `content.md` and click the pencil (Edit) icon
3. Make changes using standard Markdown syntax
4. Click "Commit changes" — Vercel redeploys automatically within ~30 seconds

### What the client can edit

- Bio texts (all three versions)
- All section titles (changing `# Projects` to `# Selected Projects` updates the nav label automatically)
- All entry content within sections
- Portrait images and captions
- Footer text and links
- Adding or removing entire sections

## 7. Design System

### Typography

- **UI typeface:** Instrument Sans (Google Fonts) — nav, footer, controls, wordcount, header pills, mobile overlays
- **Bio + entry years typeface:** ABC Diatype Mono Condensed (local OTF) — bio text, entry years in desktop panels and mobile overlays
- **Base size:** `0.9rem` (set on `:root`)
- **Line height:** `1.4` (`--line-height`)
- **Weight:** 400 throughout
- **Case:** uppercase for all UI elements
- **Letter spacing:** `0.01em`

### Colour

- **Light mode:** background `#f0f0f0`, text `#1a1a1a`
- **Dark mode:** background `#2d2c2c`, text `#EFE1E8`

All colours are CSS custom properties on `:root`. Dark mode is toggled by adding `.dark` to `<body>`, which overrides the custom properties with a smooth 0.6s crossfade.

### Dark mode — automatic

On page load, if the device hour is between 20:00 and 07:00, dark mode is applied automatically and the face enters sleeping state. Clicking the face toggles dark mode manually.

### CSS Variables

```css
:root {
  font-size: 0.9rem;
  line-height: 1.4;
  --line-height: 1.4;
  --color-text: #1a1a1a;
  --color-bg: #f0f0f0;
  --left: 12rem;
  --top: 1.5rem;
}
```

## 8. Layout

### DOM structure

```
body
├── .cursor-portrait              — fixed, pointer-events: none, z-index: 1000
├── .cursor-caption               — fixed, created by JS, z-index: 1000
├── .mobile-header                — fixed pill, mobile only, z-index: 90
├── .desktop-header               — fixed pill, desktop only, z-index: 90
├── .name-jtf                     — fixed, mobile only, z-index: 90
├── .layout
│   ├── .face                     — fixed, z-index: 100
│   └── .col-left                 — fixed, full viewport, flex column
│       ├── .name
│       ├── .bio
│       ├── .bio-controls
│       ├── .bio-spacer            — display: none (legacy)
│       ├── .sections-primary      — margin-top: auto
│       │   └── .nav-row ×N
│       │       ├── .nav-item
│       │       └── .section-panel  — position: absolute
│       ├── .sections-secondary    — margin-top: auto
│       │   └── .nav-row ×N
│       ├── .footer-spacer         — flex-shrink: 1, flex-basis: 16rem
│       └── .footer                — position: fixed on desktop
├── .section-overlay              — mobile only, z-index: 80
└── .portrait-overlay             — mobile only, z-index: 80
```

### Scrolling strategy

`.col-left` is `height: 100vh` with `overflow-y: auto`. Content fits within the viewport on most screens. When the bio expands, `margin-top: auto` on `.sections-primary` and `.sections-secondary` compresses to zero, then the column overflows and becomes scrollable.

### Section panels (desktop)

Panels are `position: absolute` inside `.nav-row` (which has `position: relative`). This means panels automatically track their nav label when the flex layout changes. Panels use `left: 280px`, `top: 0`, `max-height: calc(100vh - 3rem)` with `overflow-y: auto` (hidden scrollbar) and `padding-bottom: 5rem`. Visible panels get `z-index: 10` to stay interactive above other nav rows.

### Footer (desktop)

`position: fixed` at bottom-left. Has a pill-shaped background that appears on scroll via `.pill-visible` class (triggered when `colLeft.scrollTop > 10`). The `.footer-spacer` (flex-shrink: 1, flex-basis: 16rem) reserves bottom space in the flex column that compresses as the bio grows.

### Header pills

- `.desktop-header` — appears on scroll, wraps face + "James Taylor-Foster". Hidden on mobile.
- `.mobile-header` — appears on scroll on home page, and on scroll inside section overlays. Hidden on desktop.

### Links

All links (except `#` anchors and `mailto:`) get `target="_blank"` and `rel="noopener noreferrer"` applied globally via JS on page load. Mobile overlay links get the same treatment when content is populated.

## 9. Face State Manager

A centralised JS object controlling the face emoticon `◕‿◕`. Multiple states can be active simultaneously; the highest-priority active state renders.

### Priority order (highest first)

`blushing` → `winking` → `surprised` → `sleeping` → `default`

### States

| State | Eyes | Mouth | Blinks | Trigger |
|-------|------|-------|--------|---------|
| default | `◕ ◕` | `‿` | yes | — |
| sleeping | `◡ ◡` | `○` | no | dark mode on |
| blushing | `◡ ◡` | `‿` | no | portrait hover (desktop), portrait overlay open (mobile) |
| winking | `◕ ◠` | `‿` | no | bio copy (2s flash) |
| surprised | `> <` | `ᴗ` | yes | bio expand/collapse (500ms flash) |

### API

- `face.enter(state)` — activate a state
- `face.leave(state)` — deactivate a state
- `face.flash(state, ms)` — activate temporarily
- `face.current()` — returns highest-priority active state
- `face.render()` — updates DOM, adds state as CSS class on `.face`

### Sleep/wake on panel interaction

When a panel opens during dark mode, the face temporarily wakes (`face.leave('sleeping')`). `face._wokenByPanel` flag tracks this. On `hideAll()`, sleeping is re-entered if the flag is set.

### CSS for face states

- `.face.sleeping` — eyes spaced wider (`margin: 0.2em`), mouth offset down with breathing animation (`scale(1)` → `scale(1.2)` with `translateY(0.4em)`, 3s)
- `.face.surprised` — eyes spaced wider (`margin: 0.2em`)
- Mobile sleeping: eyes `font-size: 0.9em`, mouth `font-size: 0.7em` to match default size

## 10. JavaScript

All JS lives inside a single `DOMContentLoaded` listener in `template.html`.

### `positionNav()`

Sets the `.bio` container height to match the active bio panel's `scrollHeight`. Called on `fonts.ready`, `resize`, and after bio transitions.

### `showBio(index, animate)`

Three-step bio transition: fade out → animate height → fade in. Triggers `face.flash('surprised', 500)`. On mobile, `–` scrolls `.col-left` to top.

### Bio copy

Wordcount button shows word count, "copy bio" on hover, copies to clipboard on click. Shows "copied!" for 2s. Triggers `face.flash('winking', 2000)`. Requires HTTPS.

### `showPanel(target)` / `hideAll()`

Controls desktop section panel visibility. Staggered entry animations: `ENTRY_IN_STEP: 0.06s`, `ENTRY_OUT_STEP: 0.04s`, `HIDE_DELAY: 1800ms`. Manages `aria-expanded`, `.panel-hover` class, and face wake/sleep state.

### Nav indicator

The `–` indicator shows on: button hover, button pinned, or `.panel-hover` active. `.panel-hover` is set by `showPanel()` and removed by `hideAll()`, so the indicator disappears when the hide delay triggers.

### Scroll listener — `.col-left`

On `scrollTop > 10`:
- Toggles `.pill-visible` on footer (desktop pill background)
- Mobile: shows/hides `.mobile-header`, injects `.name-clone`
- Desktop: shows/hides `.desktop-header`, injects `.name-clone`

### Mobile overlays

On mobile, nav items (except Portrait) open `.section-overlay`. Content populated from panel innerHTML. On open, `.name-jtf` shows "James Taylor-Foster" (without pill background). On overlay scroll, the mobile header pill appears and `.name-jtf` hides. On scroll back to top, pill hides and `.name-jtf` reappears. On close, `.name-jtf` resets to "JTF" and header state restores based on main page scroll position. The overlay scroll listener is guarded with `if (!overlay.classList.contains('visible')) return` to prevent state changes during close.

### Portrait — desktop

`.entry-trigger` ("View") hover shows cursor-following image + caption. `.entry-download` provides download link. Face enters blushing state.

### Portrait — mobile

Opens `.portrait-overlay` with dynamically built images/captions. Face enters blushing on open, leaves on close. Mobile header hidden during portrait overlay. Share via Web Share API.

### Resize handling

`positionNav()` on resize. Page reloads when crossing the 768px breakpoint.

## 12. Animations

- **Fade-in cascade:** name → bio → bio-controls → nav items → footer. Base delay 0.3s, step 0.2s
- **Eye blink:** random 2–6s interval, `scaleY(0.1)` for 120ms. Skipped when `canBlink: false`
- **Dark mode crossfade:** 0.6s ease on background-color, color, link underlines
- **Bio transition:** 0.2s fade out → 0.3s height → 0.2s fade in
- **Section panels:** opacity + visibility, 0.3s ease
- **Entry cascade:** in 0.06s step top-to-bottom, out 0.04s step bottom-to-top + 400ms buffer
- **Link underline (desktop):** `::after` pseudo-element, `scaleX(0)` on hover. Hidden on mobile
- **Link underline (mobile):** CSS `text-decoration: underline`
- **Header/footer pills:** opacity 0.3s (header), background-color + box-shadow 0.3s (footer)
- **Sleeping mouth:** `scale(1)` → `scale(1.2)` with `translateY(0.4em)`, 3s ease-in-out infinite
- **Noscript fallback:** forces opacity: 1, animation: none

## 14. Mobile (≤768px)

### Layout
- `.col-left`: `position: fixed; height: 100vh; padding: 0 1.75rem`
- `.sections-primary`: `margin-top: auto`
- `.sections-secondary`: `margin-top: 2rem; margin-bottom: 7.5rem`
- `.footer`: `position: relative` (in flow, scrolls with content). `.pill-visible` overridden to `transparent`/`none`
- `.desktop-header`: `display: none`
- `.portrait-overlay`: `display: none` on desktop, `display: flex` on mobile

### Mobile header
Pill at `top: 0.6rem`. On home page: appears on scroll (`scrollTop > 10`). In section overlays: appears on scroll; `.name-jtf` shows "James Taylor-Foster" without pill on overlay open, swaps to pill on scroll, reverts on scroll back to top.

### Section overlays
Full-screen overlays. Year shown for every entry (no duplicate suppression). Year on its own line above entry text. Entry cascade fade-in on open.

### Portrait overlay
Full-screen. Share + Close buttons at bottom. Face blushes on open. Mobile header hidden during portrait view.

### Sleeping face sizing
Eyes: `font-size: 0.9em`. Mouth: `font-size: 0.7em`. Prevents sleeping characters rendering larger than default on mobile.
